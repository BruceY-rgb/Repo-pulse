# Repo-Pulse 项目计划书

## 项目范围 (Project Scope) 

### 项目目标

**Repo-Pulse** 旨在打造一个基于大语言模型（LLM）的 **SaaS 化研发效能与智能治理平台**。通过深度整合 Git 工作流，系统将海量的底层代码变更转化为高价值的语义信息，帮助开发者从信息过载中解脱，并为管理者提供量化的工程质量视图。

### 核心功能范围 (In-Scope)

- **多源数据集成适配：** 支持实时监听 GitHub/GitLab 的推送、PR 及 Issue 动态，并进行标准化处理。
- **AI 智能研判引擎：** 包含自动摘要生成、风险评估（LOW/MEDIUM/HIGH/CRITICAL）、变更分类及改进建议。
- **智能过滤与审批流：** 允许用户配置过滤规则（INCLUDE/EXCLUDE/TAG），并提供 AI 生成内容的“编辑-审批”工作流。
- **研发效能看板 (Dashboard)：** 包含 DORA 指标（部署频率、变更前置时间等）、风险趋势图及团队活跃度指标。
- **多渠道通知系统：** 支持邮件、钉钉、飞书推送，并利用 WebSocket (Socket.io) 实现应用内实时提醒。

### 项目边界 (Out-of-Scope)

- **代码托管与编辑：** 不提供代码仓库托管服务或在线 IDE 功能。
- **CI/CD 执行：** 不直接负责自动化构建、测试执行或生产部署。

## 过程模型 (Process Model)

### 混合模型：敏捷迭代开发 + 瀑布式文档驱动

- **敏捷开发 (Sprint)：** 用于应对 AI 模型调优及前端交互设计的不确定性，快速验证原型。
- **瀑布流 (Milestone)：** 严格执行软工课要求的关键评审节点，确保文档产出与代码进度同步。

### 沟通与检查机制

- **双周检制度：** 每周四、周末晚上 21:40 定期检查进度，各负责人汇报当周任务完成情况。
- **小群协作：** 开发、测试、汇报等建立专属讨论组，组长与产品介入所有小群以保持信息对齐。

## 规模估算 (Scale Estimation)

- **代码量预测：** 总 SLOC 预计约 **6000 行**（后端 3500 行，前端 2500 行）。
- **人力成本：** 8 人团队，周期 8 周，总计人力投入约 **16 人月**。
- **复杂度估算：** 核心难点在于 12 个 Next.js 功能模块的设计、Prisma 数据建模、AI 多模型抽象层以及 BullMQ 消息队列的异步可靠性。

## 风险分析 (Risk Analysis) 

- **技术风险：AI 分析结果不稳定。**
  - *对策：* 增加后端解析校验层；设计 Fallback 链，主模型失效时自动切备选模型。
- **管理风险：成员任务逾期。**
  - *对策：* 执行权重分每日扣减机制；逾期 4 天启动强制转移，“救火队员”瓜分被扣分数。
- **性能风险：高频 Webhook 阻塞。**
  - *对策：* 使用 Redis + BullMQ 进行异步解耦，配置指数退避重试策略。
